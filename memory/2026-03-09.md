## Fix Scroll-to-Top on Stream Completion (00:38 PST, 2026-03-10)
**Problem**: When streaming finished, the page re-rendered and scrolled user back to top, losing their reading position

**Root cause**: Component swap on `streamDone` change
- During streaming: rendered `StreamingText` component
- After streaming: replaced with `ParableRenderer` component
- React unmounted one component and mounted another, triggering scroll reset

**Fix**:
1. **Never swap components** — keep `StreamingText` mounted throughout streaming
2. **Add follow-up question separately** — when `streamDone=true`, append the follow-up question box below existing content
3. **No character colors during streaming** — trade-off for smooth UX (can add back later via post-processing)
4. **Same DOM structure** — parable content stays in place, only new element (follow-up) is added

Result: Smooth reading experience, no scroll jumps, no lost position. 🔥

## Fix Scroll Reset - The Real Root Cause (09:45 PST, 2026-03-10)
**Problem**: Still losing scroll position when streaming finished (even after previous fix)

**Root cause identified**: 
- When we called `setSearchParams({}, { replace: true })` to remove `?stream=true`, it triggered React Router to re-render
- Even with `replace: true`, changing URL params causes component updates
- ANY state change after streaming = potential scroll reset

**The actual fix**:
- **Don't transition out of streaming mode at all**
- Keep the `?stream=true` URL param
- Don't call `setSearchParams()`
- Don't try to create synthetic lesson objects
- Just stay in streaming render mode with `streamDone=true`

**Trade-off**: URL still shows `?stream=true` after completion, but:
- Scroll position is perfectly preserved ✓
- All content displays correctly ✓
- When user navigates away or refreshes, it loads normally ✓

Result: Zero state changes after streaming = zero scroll resets. Simple is better. 🔥

## Fix Generate Next Button (17:33 PST, 2026-03-10)
**Problem**: Clicking "Generate Day X" button after streaming finishes didn't start streaming the next lesson - it returned to same page

**Root cause**: 
- When navigating from Day 5 → Day 6 with `?stream=true`, the URL changed but streaming state didn't reset
- Old state from Day 5 (streamDone=true, streamParable="...", etc.) persisted
- Component thought it was still showing completed Day 5 content
- React Router updated the URL params but didn't reset component state

**The fix**:
- Added state reset in the existing `useEffect` that runs on `[seriesKey, sortOrder]` changes
- Resets all streaming state: streamPhase, streamTitle, streamParable, streamFollowUpQuestion, streamImage, streamDone
- Closes existing EventSource connections and polling intervals
- Now when you navigate to a new lesson, everything starts fresh

**Result**: "Generate Day X+1" button now properly navigates to next day and starts streaming new content. 🔥
