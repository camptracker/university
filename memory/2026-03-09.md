## Character Color Fix (00:08 PST, 2026-03-10)
**Issue**: Character names and dialogue weren't being colored

**Root cause**: Previous implementation tried to inject HTML into markdown text, which was fragile and broken

**Fix**: Rewrote `ParableRenderer.tsx` to use ReactMarkdown's custom components:
- `strong` component checks if bold text is a character name → colors it and tracks "last character"
- `em` component checks if italic text starts with quote → uses last character's color
- Much cleaner and reliable approach

## Stream Completion Without Refresh (00:24 PST, 2026-03-10)
**Goal**: No layout shift or jitter when streaming completes

**Changes**:
1. **Don't reload after streaming** — keep streamed content in state, just set `streamDone=true`
2. **Transition smoothly** — switch from `StreamingText` to `ParableRenderer`/`ReactMarkdown` when done
3. **Identical placeholders** — skeleton loaders match real content structure:
   - Image placeholder (aspect-ratio 1:1)
   - Toggle buttons
   - Multiple skeleton lines (~6 lines for text)
4. **Show navigation** — bottom nav appears after stream completes
5. **All loading states unified** — normal loading, waiting for gen, and streaming all use same skeleton structure

Result: Zero layout shift, no page refresh, smooth experience. 🔥
