## Fix Scroll-to-Top on Stream Completion (00:38 PST, 2026-03-10)
**Problem**: When streaming finished, the page re-rendered and scrolled user back to top, losing their reading position

**Root cause**: Component swap on `streamDone` change
- During streaming: rendered `StreamingText` component
- After streaming: replaced with `ParableRenderer` component
- React unmounted one component and mounted another, triggering scroll reset

**Fix**:
1. **Never swap components** ŌĆö keep `StreamingText` mounted throughout streaming
2. **Add follow-up question separately** ŌĆö when `streamDone=true`, append the follow-up question box below existing content
3. **No character colors during streaming** ŌĆö trade-off for smooth UX (can add back later via post-processing)
4. **Same DOM structure** ŌĆö parable content stays in place, only new element (follow-up) is added

Result: Smooth reading experience, no scroll jumps, no lost position. ­¤öź
